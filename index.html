JavaScript

Основні концепції JavaScript: змінні, функції, об'єкти, масиви, оператори та умовні конструкції.

Функції - це блоки коду, які можна використовувати багаторазово, для виконання певних завдань.

Обєкти - це структури даних, які містять властивості та методи для роботи з даними.

Масиви - це тип об'єктів, які зберігають послідовність елементів, що можуть бути будь-якого типу.

Оператори - це символи або ключові слова, які використовуються для виконання певних операцій.

Умовні конструкції - це конструкції, які дозволяють виконувати код, якщо певна умова справджується.

ООП в JavaScript: класи, об'єкти, наслідування, поліморфізм.

Класи - це шаблон для створення об'єктів, що містить властивості та методи.

Об'єкти - це екземпляри класів, які містять властивості та методи, що визначені в класі.

Наслідування - це процес, за яким клас може успадковувати властивості та методи від іншого класу.

Поліморфізм - це здатність об'єктів одного класу використовуватися замість об'єктів іншого класу.

Асинхронність в JavaScript: callback-функції, Promise, async / await.

Callback-функції - це функції, які передаються як аргументи до інших функцій та виконуються після виконання основної функції.

Promises - це інший підхід до асинхронного програмування. Вони повертають обіцянку в результаті виконання асинхронної операції, яку можна обробити за допомогою методів then() та catch().

У JavaScript асинхронний код виконується за допомогою концепції callback-функцій, обіцянок (promises) та async/await. Callback-функції передаються викликаній функції, яка запускається асинхронно. Коли функція завершується, вона викликає callback-функцію з результатом виконання.

Event loop - це механізм, який дозволяє JavaScript оброблювати події в асинхронному режимі та уникати блокування виконання коду.

Робота з DOM: додавання / видалення елементів, зміна стилів та атрибутів.

DOM - це структура, яка представляє HTML-документ у вигляді дерева, де кожен елемент є вузлом.

document - це об'єкт, який представляє HTML-документ та дозволяє звертатися до його вузлів.

Селектори - це рядки, які використовуються для вибору HTML-елементів на сторінці, наприклад, за класом, id або тегом.

Маніпуляція з вузлами - це процес додавання, видалення та зміни HTML-елементів на сторінці за допомогою JavaScript.

Ajax та робота з API: взаємодія з сервером, отримання та передача даних.

AJAX - це технологія, яка дозволяє взаємодіяти з сервером без перезавантаження сторінки.

XMLHttpRequest - це об'єкт, який дозволяє виконувати запити до сервера та отримувати відповіді.

Методи запитів - це HTTP-методи, які використовуються для взаємодії з сервером, такі як GET, POST, PUT та DELETE.

Обробка відповідей - це процес обробки відповіді від сервера та виконання необхідних дій зі змістом відповіді.

Конструктор

- це спеціальна функція в JavaScript, яка дозволяє створювати об'єкти. Вона використовується для ініціалізації нового об'єкта, встановлення його властивостей та методів, а також для передачі параметрів при створенні об'єкту.

У JavaScript конструктори використовуються для створення класів, які можна використовувати для створення багатьох об'єктів з однаковою структурою та функціональністю. Конструктор можна викликати за допомогою ключового слова "new", яке створює новий екземпляр класу з властивостями та методами, встановленими в конструкторі.

Наслідування

- це механізм, який дозволяє створювати нові класи на основі існуючих класів з використанням властивостей та методів цих класів. Клас, який надає свої властивості та методи для нового класу, називається батьківським класом або суперкласом, а новий клас, який успадковує ці властивості та методи, називається дочірнім класом або підкласом.

Прототипне ООП

- це парадигма програмування, в якій об'єкти засновані на "прототипах", тобто на інших об'єктах, які служать в якості зразків або шаблонів для створення нових об'єктів. Замість класів, як у класичному ООП, у прототипному ООП використовуються об'єкти з визначеними властивостями, які можуть бути унаслідовані та змінені.

У прототипному ООП, кожен об'єкт має посилання на інший об'єкт, який використовується як його прототип. Якщо властивість або метод не знайдено в об'єкті, JavaScript знаходить його в прототипі та повертає значення. Це дозволяє створювати нові об'єкти, які наслідують властивості та методи від інших об'єктів, що зменшує дублювання коду та полегшує розробку.

Патерни в JavaScript

- це шаблони проектування, які дозволяють вирішувати певні задачі з використанням певних структур коду. Ось декілька популярних паттернів в JavaScript:

Фабричний метод (Factory Method): Цей паттерн використовується для створення об'єктів без явного вказання їх типу або класу. Він дозволяє створювати об'єкти на основі умов, які визначаються під час виконання програми.

Адаптер (Adapter): Цей паттерн використовується для перетворення інтерфейсів об'єктів таким чином, щоб вони могли працювати разом. Він дозволяє використовувати об'єкти, які несумісні за інтерфейсом, разом.

Наблюдатель (Observer): Цей паттерн використовується для створення механізму, який дозволяє об'єктам спостерігати за змінами інших об'єктів та реагувати на них.

Стратегія (Strategy): Цей паттерн використовується для визначення набору алгоритмів та вибору одного з них в залежності від контексту. Він дозволяє керувати поведінкою об'єкта, не змінюючи його класу.

Цепочка відповідальності (Chain of Responsibility): Цей паттерн використовується для створення ланцюжка об'єктів, які можуть обробляти запити в порядку їх надходження. Він дозволяє реалізувати гнучку обробку запитів в залежності від умов.

Декоратор (Decorator): Цей паттерн використовується для додавання нової поведінки об'єкту без зміни його класу. Він дозволяє динамічно змінювати функціональність об'єкта, додаючи нові метод
